# OCTO Loan Docs

Published lender packages generated by [octo-loan-tool](https://github.com/djackson74/octo-loan-tool).

Each publish run creates a dated folder with markdown source, Word (`.docx`), and PDF copies plus a `manifest.json` describing the live dashboard inputs used to size the facility.

## Layout

```
loan/
  latest.json                 # pointer to most recent publish
  YYYY-MM-DD/
    HH-MM-SSZ/
      manifest.json           # run metadata + file index
      capex-quotes.json       # locked vendor CAPEX split (copy)
      markdown/               # source .md files
      docx/                   # Word exports (pandoc)
      pdf/                    # PDF exports (pandoc or md-to-pdf)
      markdown-revised/       # CHRITIC-revised markdown (optional)
```

Revised lender packages (baseline definition, logic chains, Schedule D, etc.) are written to `pdf/` as `*_Revised_YYYY-MM-DD.pdf` plus supporting schedules. See `manifest.json` → `revised` block when present.

## Publish

From `loan-tool`:

```bash
npm run publish
```

This runs `docs` (live dashboard pull → markdown), then exports to `../octo-loan-docs/loan/...`.

Options:

```bash
node publish-docs.js --skip-generate          # reuse existing loan-tool/output/*.md
node publish-docs.js --docs-repo /path/to/octo-loan-docs
npm run revised   # from loan-tool — CHRITIC-revised PDFs into latest publish pdf/
```

## Requirements

| Format | Tool | Install |
|--------|------|---------|
| DOCX | [Pandoc](https://pandoc.org/) | `winget install JohnMacFarlane.Pandoc` |
| PDF | Pandoc PDF engine or `md-to-pdf` | `npm install` in loan-tool (md-to-pdf fallback) |

Optional: LibreOffice for higher-fidelity PDF via DOCX (`soffice --convert-to pdf`).

## OCTO on Pi?

Not required. Document generation runs on your dev machine (Node + Pandoc). The Pi workspace kit is for local OCTO Master Control, not lender doc export.
